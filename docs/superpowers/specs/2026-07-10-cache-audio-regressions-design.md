# 缓存音频回归修复设计

## 目标

修复截图确认的三项回归：FLAC 边缓存 PCM 配置失败导致无声、缓存页搜索框显示为白色、完整缓存实际播放时诊断仍显示等待授权。

## 根因与修复

### FLAC 边缓存播放

FLAC 描述继续使用 WebCodecs 规范要求的 `fLaC` 标记和 STREAMINFO 元数据块。`AudioDecoder.isConfigSupported()` 返回的配置是浏览器已检查并规范化的配置，后续 `configure()` 必须使用该返回值，不能重新使用原始配置。支持检查失败或不支持时停止创建解码器并保留明确失败状态。

### 缓存页输入框

缓存页搜索框不再使用无法为十六进制 CSS 变量正确生成透明度的 `bg-background/50`，改用明确的深色背景并声明暗色控件配色，保证 Chromium、WebView2 和系统表单样式一致。

### 完整缓存诊断与播放意图

完整缓存的可听判断以真实音频元素为准。原生 Blob 音源存在、媒体数据可用、元素未暂停、未静音且音量大于零时，应显示“正在发声”，不再要求可能滞后的 `fullLocalReady`。

播放元素实际启动成功后，如果当前曲目或队列项与待处理播放意图一致，即使服务端 revision 尚未追平，也消费该意图，避免持续显示“正在准备音源”或“等待音频授权”。

## 边界

- 不更换解码库。
- 不修改同步播放时钟、漂移校正或缓存数据格式。
- 不修改后端接口。
- 仅增加覆盖三个截图问题的定向回归测试。
