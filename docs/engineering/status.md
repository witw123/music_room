# 项目状态

最后更新：`2026-03-28`

## 总览

### 已完成

- `Monorepo` 仓库结构已落地。
- `Next.js` 前端工作台可运行。
- `NestJS` 服务端 API 与 WebSocket 可运行。
- `packages/shared` 已定义核心共享模型。
- 房间、队列、播放器、歌单基础闭环可用。
- Redis 房间快照广播已接入。
- Redis 最近活跃房间恢复已接入。
- Redis 房间注册表与按房间恢复接口已接入。
- WebSocket 订阅后立即下发当前房间快照。
- Prisma 持久化已接入，并保留无库降级模式。
- IndexedDB 整曲缓存与分片缓存已接入。
- 分片可用性广播和前端 P2P 状态面板已接入。
- WebRTC DataChannel chunk 请求/接收 PoC 已接入。
- chunk hash 校验与整曲 hash 校验已接入。
- 当前曲目优先拉取与下一首预取已接入。
- socket 自动重连与房间重订阅已接入。
- 服务端房间服务、控制器、网关测试已接入。
- 前端 P2P 纯逻辑测试已接入。
- `typecheck`、`build`、`test` 已通过。
- Docker Compose 与双 Dockerfile 已落地。

### 进行中

- Redis 房间状态缓存和更完整的重连补偿。
- WebRTC 分片调度、超时重试和重组校验增强。
- WebSocket 事件协议继续收敛。
- 房主/成员权限进一步细化。

### 未完成

- 完整 WebRTC 音频分片传输。
- IndexedDB 分片缓存完善和播放前预取策略增强。
- TURN 中继与 NAT 失败兜底。
- 协作歌单。
- 观测指标、告警和错误追踪。
- 前端组件测试与 E2E 自动化。

## 最近优先级

1. Redis 房间状态缓存和更稳的重连恢复。
2. DataChannel 传输稳定性、完整分片重组和更可靠的 peer 调度。
3. IndexedDB 分片预取、校验和缓存回收。
4. 前端组件测试与 E2E 主流程自动化。
5. 观测与部署文档继续补齐。

## 暂不承诺

- 正版版权曲库接入。
- 服务端音频转发。
- 原生移动端。
- 跨设备媒体断点同步。
