# 整体架构

## 核心组件

- `apps/web`: 承担房间 UI、播放器、歌单、上传和 P2P 逻辑
- `apps/server`: 承担 REST API、WebSocket 信令、房间状态与元数据管理
- `Redis`: 房间在线态、播放状态缓存、实时广播辅助
- `PostgreSQL`: 游客身份、歌单、曲目元数据、队列历史
- `WebRTC`: 客户端间音频分片传输

## 数据与控制流

1. 用户通过 Web 端创建游客会话
2. Web 端调用服务端创建房间并建立 WebSocket
3. 房主导入本地歌曲并广播资源可用性
4. 成员通过 WebRTC 与房主或其他成员交换分片
5. 房主通过 WebSocket 发送播放控制事件
6. 所有客户端根据快照和本地缓冲状态同步播放

## 边界

- 服务端不托管音频文件
- 客户端缓存命中优先于重新下载
- P2P 失败时不引入服务端媒体中转
