# 播放同步

## 权威模型

- 房主维护权威播放状态
- 服务端负责广播房主指令和快照
- 成员不自行决定播放节奏，只执行校准

## 核心字段

- `status`
- `currentTrackId`
- `positionMs`
- `startedAt`
- `queueVersion`

## 同步流程

1. 房主发出播放或暂停指令
2. 服务端广播播放快照
3. 成员结合本地时间和快照时间校准当前位置
4. 若本地关键 chunk 缺失则先缓冲再进入播放

## 误差控制

- 周期性发送轻量 sync ping
- 成员若误差超过阈值则微调 position
- 误差持续过大时强制 seek 到房主位置
