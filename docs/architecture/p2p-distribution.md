# P2P 分发设计

## 文件导入

- 客户端读取本地音频文件
- 解析标题、歌手、专辑、封面和时长
- 计算文件哈希作为资源标识的一部分
- 将文件切分为固定大小 chunk

## 资源分发

- 房主先声明自己持有全量 chunk
- 成员加入后先拉取当前播放歌曲的起播窗口
- 已下载成员同步声明自己持有的 chunk 区间
- 新成员优先并行从多个节点获取分片

## 缓存策略

- 分片缓存在 IndexedDB
- 以 `trackId + fileHash + chunkIndex` 作为唯一键
- 当前歌曲优先、下一首预取、其他歌曲低优先级补齐

## 失败处理

- 单个 peer 断开后切换到其他可用上游
- 分片校验失败则丢弃并重新拉取
- 所有上游失败时播放器进入 buffering 并提示用户
