# P2P 资产分发

最后更新：`2026-07-15`

## 设计目标

- 服务端不保存音频文件
- 房主和已缓存成员通过 WebRTC DataChannel 分发资产分片
- 当前播放资产优先进入 IndexedDB，供分段 Opus 引擎读取
- 原始资产下载和手动缓存独立于播放输出

## 分片来源

- `live_upload`：上传者本地资产
- `local_cache`：成员已经验证并缓存的资产

公告包含 `trackId`、资产 hash、分片范围、分片 hash 和 owner peer。接收端先校验内容 hash，再写入 IndexedDB；传输调度器按当前资产、前向窗口和后台资产分配优先级。

## 调度原则

1. 当前播放资产最高优先级
2. 启动阶段优先填充当前位置附近的连续分段
3. 正常播放维持前向资产窗口
4. 暂停时可继续整理当前资产，但不改变播放会话
5. 原始资产手动缓存只负责资源保存、下载和导出

DataChannel 失败时，分段资产会等待传输恢复；播放引擎保持现有 AudioContext、destination 和 RTP Track，并输出静音，不切换到另一种播放引擎。
