# 房间流程

## 房间生命周期

1. 房主进入首页并创建房间
2. 系统创建游客会话与房间快照，返回房间码
3. 房主导入本地歌曲，客户端生成哈希、元数据和资源声明
4. 成员通过房间码加入，获取房间状态、成员列表和当前队列
5. 成员建立 WebRTC 连接并拉取当前歌曲的关键分片
6. 全房间依据房主的播放指令同步播放
7. 房主离开时房间结束

## 权限规则

- 房主可播放、暂停、切歌、跳转、删除队列项、调整顺序
- 成员可点歌、查看队列、收藏歌单、参与协作歌单
- 房主可关闭成员点歌、限制排序权或设置房间私密性

## 交互重点

- 队列更新必须是全房间实时一致的
- 成员加入时优先为当前歌曲建立起播能力
- 成员可在不中断房间的情况下补齐后续歌曲缓存
