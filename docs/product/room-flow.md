# 房间流程

最后更新：`2026-07-23`

## 房间生命周期

1. 用户登录或注册
2. 创建房间，或通过房间码加入房间
3. 客户端订阅房间并建立 peer
4. 房主或成员导入本地音频，或从已启用的 provider 导入曲目
5. 浏览器注册曲目并生成本地播放资产
6. 用户把曲目加入共享队列
7. 用户点击播放
8. 源端由分段 Opus 引擎输出 WebRTC RTP Opus，成员端绑定单一远端 audio stream
9. 成员可以点歌、重排、切歌、暂停和 seek
10. 用户暂离房间时保留实时成员关系，返回房间后继续当前会话
11. 普通成员离开后从成员列表移除，房主离开后保留离线房主身份
12. 房主主动删除房间后，所有媒体会话释放

## 播放控制

- 播放状态以房间共享状态为准
- 所有成员围绕同一条房间时间线播放
- 成员加入、presence 更新、普通快照和音量变化不会重建媒体 Track
- 切歌、source peer 变化或房间退出才会改变或释放媒体会话

## 个人曲库和本地资产

- 房主或成员可以导入本地文件
- 上传时浏览器在本地生成原始资产和分段 Opus 播放资产
- IndexedDB 用于保存当前用户自己上传或 provider 导入的歌曲，刷新或重新进入后恢复个人曲库
- 房间不提供缓存下载、缓存导出或成员间音频资产传输
- 实际音频文件不上传到服务端
- provider 临时音频不会写入房间状态，导入后仍由当前用户浏览器持有
