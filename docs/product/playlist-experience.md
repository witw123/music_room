# 歌单能力现状

最后更新：`2026-07-07`

## 当前结论

歌单后端能力仍然存在，当前房间体验的主入口是共享队列。

当前保留的能力有：

- 从当前房间队列保存为歌单
- 重命名歌单
- 删除歌单
- 将歌单重新导回房间队列

## 当前 UI 状态

- 房间页默认已移除歌单区域
- 房间工作区当前默认保留队列、曲库、缓存和成员
- 因此歌单在产品层面属于“保留能力”，当前主路径仍是房间共享队列

## 为什么这么调整

当前版本的优先级已经转向：

- 多人同步播放稳定性
- 渐进式本地播放
- 连接与缓存诊断
- 桌面端 / 移动端发布链路稳定

歌单协作在本阶段属于次级体验。

## 未来如果恢复歌单入口

更合理的方向是：

- 把歌单作为房间外的整理能力
- 房间内仍以共享队列为主
- 保留“从队列沉淀歌单”和“把歌单导入队列”的双向转换
