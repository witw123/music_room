# 歌单能力现状

最后更新：`2026-07-15`

## 当前结论

歌单后端能力仍然存在，当前房间体验的主入口是共享队列。

保留能力：

- 从当前房间队列保存为歌单
- 重命名歌单
- 删除歌单
- 将歌单重新导回房间队列

## 当前 UI 状态

- 房间页默认不显示独立歌单区域
- 房间工作区当前默认保留 `Queue`、`Library`、`Members`
- 歌单属于房间外的整理能力，不参与媒体源选择或资产传输

## 产品优先级

当前版本优先保证：

- 多人同步播放稳定性
- 单一 Segmented Opus/WebRTC 媒体链路
- 个人上传曲库在浏览器刷新后的恢复
- 成员与媒体诊断
- 桌面端和移动端发布链路稳定

## 未来恢复入口时的边界

- 歌单作为房间外的整理能力
- 房间内仍以共享队列为主
- 保留“从队列保存歌单”和“把歌单导入队列”的双向转换
- 不把歌单设计成缓存下载、媒体源切换或成员间音频共享入口
